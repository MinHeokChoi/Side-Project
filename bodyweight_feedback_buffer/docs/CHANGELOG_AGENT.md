# Changelog Agent

이 문서는 기능 업데이트를 할 때마다 `docs/CHANGELOG.md`를 함께 갱신하기 위한 에이전트 지침입니다.

## 목적

사용자에게 보이는 변경사항을 놓치지 않고 배포 기록으로 남깁니다. `docs/CHANGELOG.md`는 App Store 릴리즈 노트의 원본으로 사용할 수 있어야 합니다.

## 언제 업데이트할까

다음에 해당하면 `docs/CHANGELOG.md`를 업데이트합니다.

- 새 기능이 추가됨
- 기존 화면, 문구, 흐름, 계산 방식이 바뀜
- 사용자가 겪는 버그가 수정됨
- 온보딩, 접근성, 빈 상태, 오류 상태처럼 사용 경험이 개선됨

다음만 바뀐 경우에는 보통 업데이트하지 않습니다.

- 내부 리팩터링
- 코드 포맷팅
- 주석 정리
- 테스트 코드만 변경
- 빌드 설정이나 개발 도구 변경

## 작성 위치

기본적으로 다음 섹션에 기록합니다.

```md
## 1.0.1 - 예정
```

항목은 성격에 따라 아래 중 하나에 넣습니다.

- `Added`: 새로 생긴 기능
- `Changed`: 기존 기능, 화면, 문구, 흐름의 변경
- `Fixed`: 버그 수정

## 작성 방식

- 한국어로 작성합니다.
- 사용자가 체감하는 결과를 중심으로 씁니다.
- 한 항목은 한 줄로 짧게 씁니다.
- 구현 세부사항, 파일명, 내부 클래스명은 쓰지 않습니다.
- App Store 릴리즈 노트로 옮겨도 어색하지 않게 씁니다.

좋은 예:

```md
### Added
- 운동 기록 화면에서 최근 입력값을 더 쉽게 다시 사용할 수 있도록 개선

### Changed
- 피드백 문구를 더 짧고 명확하게 조정

### Fixed
- 특정 조건에서 버퍼 계산 결과가 바로 갱신되지 않던 문제 수정
```

피해야 할 예:

```md
- Update WorkoutViewModel state handling
- Refactor FeedbackCalculator
- Fix typo
```

## 작업 순서

1. 기능 변경 내용을 구현합니다.
2. 변경이 사용자에게 보이는지 판단합니다.
3. 사용자에게 보이면 `docs/CHANGELOG.md`의 예정 버전 섹션에 항목을 추가합니다.
4. 이미 비슷한 항목이 있으면 중복하지 말고 기존 항목을 더 정확하게 다듬습니다.
5. 최종 보고에서 changelog 업데이트 여부를 함께 언급합니다.
